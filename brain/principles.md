# Project Principles

## Foundational

- **prove-it-works** — "It compiles" is not verification. Every change must describe how to prove it works.
- **exhaust-the-design-space** — For architectural decisions, sketch 2-3 approaches before committing.
- **foundational-thinking** — Name key types/schemas before logic. Data shapes drive design.

## Process

- **ship-in-small-phases** — Prefer 8-10 small phases over 3-4 large ones. Each phase independently verifiable.
- **async-first** — Plans should be executable by agents without real-time human interaction.
